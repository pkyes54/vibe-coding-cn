# Step 2: 系统分析与可视化建模

## 任务目标
对问题进行深入分析，建立系统的可视化模型，包括架构图、依赖关系等。

## 输入
- Step 1 生成的问题描述
- 相关技术文档

## 输出要求

### 1. 系统架构分析
- 识别核心模块和组件
- 分析模块间的交互关系
- 识别数据流和控制流

### 2. 可视化模型
创建以下图表：
- **系统架构图**: 展示整体结构
- **模块依赖图**: 展示模块间依赖
- **数据流图**: 展示数据流动路径
- **时序图**: 展示关键交互流程

### 3. 技术选型分析
- 评估不同技术方案的优劣
- 基于胶水工程原则优先复用
- 推荐最佳技术栈

### 4. 风险识别
- 识别潜在的技术风险
- 识别实施难点
- 提出风险缓解措施

## Canvas 白板使用
如果项目规模 > 5 个模块，使用 Canvas 白板创建可视化架构：
- 创建架构图节点
- 建立节点间连接
- 添加注释和说明

## 输出格式
```markdown
## 系统架构分析
[详细分析内容]

## 可视化模型
### 系统架构图
[Mermaid 或 ASCII 图]

### 模块依赖图
[Mermaid 或 ASCII 图]

## 技术选型
[技术方案对比和推荐]

## 风险识别
[风险列表和缓解措施]
```

## 注意事项
- 保持架构简洁，避免过度设计
- 优先考虑现有库和组件的复用
- 确保模型可被理解和实现
