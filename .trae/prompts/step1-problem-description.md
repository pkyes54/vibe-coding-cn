# Step 1: 问题描述

## 任务目标
根据用户需求，生成清晰、完整的问题描述文档。

## 输入
- 用户需求描述
- 项目背景信息

## 输出要求
生成以下内容：

### 1. 问题概述
- 简明扼要地描述要解决的问题
- 问题的背景和重要性
- 问题的范围和边界

### 2. 需求分析
- 功能需求（必须实现的功能）
- 非功能需求（性能、安全、可维护性等）
- 约束条件（技术、时间、资源等）

### 3. 成功标准
- 如何判断问题已解决
- 可量化的指标
- 验收标准

### 4. 相关信息
- 涉及的技术栈
- 可能的解决方案方向
- 需要参考的资源

## 注意事项
- 使用清晰、简洁的语言
- 避免模糊不清的描述
- 确保问题可被理解和验证
